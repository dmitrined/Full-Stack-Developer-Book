# SQL vs NoSQL: Выбор правильной БД 💾

Выбор базы данных зависит от структуры данных, требований к масштабируемости и консистентности.

## 1. Реляционные БД (SQL)
**Примеры:** PostgreSQL, MySQL, MS SQL, Oracle.

### Основные характеристики:
- **Структура:** Данные хранятся в таблицах со строгими схемами.
- **Связи:** Использование JOIN для объединения данных из разных таблиц.
- **Транзакции:** Полная поддержка ACID.
- **Масштабирование:** В основном вертикальное (увеличение мощности сервера).

### Когда использовать:
- Когда важна целостность данных (финансовые системы).
- Когда данные имеют четкую структуру и предсказуемые связи.

## 2. Нереляционные БД (NoSQL)
**Примеры:** MongoDB (Документная), Redis (Key-Value), Cassandra (Колоночная), Neo4j (Графовая).

### Основные характеристики:
- **Структура:** Гибкая схема (Dynamic Schema). Данные могут быть JSON-подобными документами.
- **Связи:** Обычно данные денормализованы (вложены друг в друга), JOIN отсутствуют.
- **Масштабирование:** Горизонтальное (добавление новых серверов в кластер).

### Когда использовать:
- Высокая нагрузка и необходимость быстрого масштабирования.
- Неструктурированные или постоянно меняющиеся данные.
- Кэширование (Redis).

## 3. ACID vs BASE

| Характеристика | ACID (SQL) | BASE (NoSQL) |
| :--- | :--- | :--- |
| **Consistency** | Strong (Строгая) | Eventual (В конечном счете) |
| **Availability** | Высокая | Очень высокая |
| **State** | Упорядоченное | Мягкое состояние (Soft state) |

## 4. CAP-теорема
Утверждает, что распределенная система может обеспечить только 2 из 3 свойств:
- **C** (Consistency) — Согласованность.
- **A** (Availability) — Доступность.
- **P** (Partition Tolerance) — Устойчивость к разделению.
