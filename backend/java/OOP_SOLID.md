# OOP & SOLID Principles 🏗️

Объектно-ориентированное программирование — это способ организации кода, имитирующий реальный мир.

## 1. Четыре столпа ООП

1.  **Инкапсуляция** — сокрытие внутренней реализации и предоставление доступа через публичный интерфейс (getters/setters).
2.  **Наследование** — механизм создания новых классов на основе существующих для повторного использования кода.
3.  **Полиморфизм** — возможность использовать объекты разных типов через единый интерфейс (переопределение методов `@Override`).
4.  **Абстракция** — выделение значимых характеристик объекта, игнорируя детали (интерфейсы и абстрактные классы).

## 2. SOLID

| Буква | Принцип | Суть |
| :--- | :--- | :--- |
| **S** | Single Responsibility | У класса должна быть только одна причина для изменения. |
| **O** | Open/Closed | Программные сущности открыты для расширения, но закрыты для модификации. |
| **L** | Liskov Substitution | Объекты в программе должны быть заменяемыми на экземпляры их подтипов без изменения правильности программы. |
| **I** | Interface Segregation | Много специализированных интерфейсов лучше, чем один универсальный. |
| **D** | Dependency Inversion | Зависимость на абстракциях, а не на конкрециях. |

## 3. DRY, KISS, YAGNI
- **DRY** (Don't Repeat Yourself) — не дублируй логику.
- **KISS** (Keep It Simple, Stupid) — делай проще.
- **YAGNI** (You Ain't Gonna Need It) — не пиши код "на будущее", который не нужен сейчас.
