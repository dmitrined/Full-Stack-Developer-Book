# Java Core: Основы (Core Basics) 🧠

Этот раздел посвящен базовым концепциям JVM, памяти и основным контрактам объектов.

## 1. Память: Stack vs Heap

JVM разделяет оперативную память на две основные области:

| Характеристика | Stack (Стек) | Heap (Куча) |
| :--- | :--- | :--- |
| **Что хранится** | Примитивы, ссылки на объекты, вызовы методов. | Сами объекты, массивы. |
| **Доступность** | Поток-локальный (у каждого свой). | Общий для всего приложения. |
| **Жизненный цикл** | LIFO. Удаляется при выходе из метода. | Управляется Garbage Collector. |
| **Скорость** | Очень быстро. | Медленнее (требует управления). |

## 2. String Pool (Пул строк)

Строки в Java неизменяемы (**Immutable**).
- Пул строк — это специальная область в Heap, где хранятся уникальные строковые литералы.
- `String s = "Hello";` — ищет "Hello" в пуле.
- `String s = new String("Hello");` — создает НОВЫЙ объект в куче, игнорируя пул (выгодно использовать `.intern()`).

## 3. Контракт Equals & HashCode

Это база для работы с коллекциями (особенно `HashMap`, `HashSet`).

*   **Если `obj1.equals(obj2) == true`, то `obj1.hashCode() == obj2.hashCode()` обязан быть true.**
*   Если хеш-коды разные, объекты точно разные.
*   Если хеш-коды одинаковые, объекты МОГУТ быть одинаковыми (коллизия).

## 4. Передача параметров

В Java **всегда** передача идет по значению (**Pass-by-value**).
- Для примитивов копируется значение.
- Для объектов копируется **значение ссылки** (указатель). Это значит, что вы можете изменить состояние объекта, но не можете заставить исходную ссылку указывать на другой объект внутри метода.
