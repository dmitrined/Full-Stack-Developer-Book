# Java Collections Framework 📚

Иерархия структур данных для хранения и манипулирования группами объектов.

## 1. Иерархия интерфейсов

- **Iterable** <- **Collection**
  - **List** (Упорядоченный список, дубликаты разрешены)
    - `ArrayList` (на базе массива, быстрый доступ по индексу).
    - `LinkedList` (связный список, быстрая вставка/удаление в начало/конец).
  - **Set** (Уникальные элементы)
    - `HashSet` (на базе HashMap, порядок не гарантирован).
    - `LinkedHashSet` (сохраняет порядок вставки).
    - `TreeSet` (сортированный порядок, на базе красно-черного дерева).
  - **Queue** (Очереди, FIFO)
    - `PriorityQueue` (сортировка по приоритету).
    - `Deque` (двусторонняя очередь).

## 2. Map (Пары Ключ-Значение)
*Не наследуется от Collection!*
- **HashMap** — Сложность O(1) в среднем. 
  - *Internal*: Массив бакетов. При коллизии — связный список, после 8 элементов — дерево (Treeify).
- **TreeMap** — Сортировка по ключам, O(log n).
- **LinkedHashMap** — Сохраняет порядок вставки.

## 3. Выбор коллекции

| Задача | Решение |
| :--- | :--- |
| Нужно быстро читать по индексу | `ArrayList` |
| Нужно хранить только уникальные значения | `HashSet` |
| Нужна сортировка | `TreeSet` / `TreeMap` |
| Потокобезопасная мапа | `ConcurrentHashMap` |
