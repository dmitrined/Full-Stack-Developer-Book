# QA & Testing Theory 🧪

Основы обеспечения качества программного обеспечения.

## 1. Виды тестирования

| По объекту | По запуску | По доступу к коду |
| :--- | :--- | :--- |
| **Функциональное** (что делает) | **Ручное** (Manual) | **Black Box** (не видим код) |
| **Нефункциональное** (нагрузка, UI) | **Автоматизированное** | **White Box** (видим код/структуру) |

## 2. Пирамида тестирования (Testing Pyramid)
Концепция, говорящая о том, что тестов низкого уровня должно быть больше.
- **Unit Tests** (Основание) — Тестирование отдельных методов. Быстрые, дешевые.
- **Integration Tests** — Проверка взаимодействия модулей/БД.
- **E2E (End-to-End) Tests** (Вершина) — Прохождение сценария как пользователь. Медленные, дорогие.

## 3. Жизненный цикл бага (Bug Lifecycle)
`New` -> `Assigned` -> `In Progress` -> `Fixed` -> `Pending Retest` -> `Verified` -> `Closed`.

## 4. Тест-дизайн (Техники)
- Эквивалентное разделение (Boundary Values).
- Граничные значения.
- Таблица принятия решений.
