
## 🧠 Что такое Zod?

**Zod** — это библиотека валидации схем, ориентированная на TypeScript. Она позволяет декларативно определять схемы данных и выполнять проверку данных во время выполнения с полной поддержкой вывода типов TypeScript.

___

## 💡 Почему Zod?

-   ✅ **Безопасность типов**: автоматически выводит статические типы из схем.
-   ✅ **Без генерации кода**: в отличие от библиотек вроде Yup или Joi, Zod не требует сборки или генерации кода.
-   ✅ **Простой API**: функциональный, цепочный и выразительный.
-   ✅ **Повторное использование схем**: одну и ту же схему можно использовать и для проверки данных во время выполнения, и для вывода типов в TypeScript.

___

## 🧩 Основные концепции

### 1\. Схемы

Схемы Zod определяют, как должны выглядеть допустимые данные.

```
const userSchema = z.object({
  name: z.string(),
  age: z.number().min(18),
});
```

Это определяет объект с:

-   обязательным строковым `name`
-   обязательным числовым `age`, минимум 18

___

### 2\. Парсинг и валидация

Zod предлагает два основных метода:

-   `.parse(data)` Бросает ошибку, если данные недопустимы.
    
-   `.safeParse(data)` Возвращает объект результата: `{ success: true; data }` или `{ success: false; error }`
    

```
const result = userSchema.safeParse(input);
if (!result.success) {
  console.log(result.error.issues); // подробные ошибки
}
```

___

### 3\. Уточнение и кастомная логика

Zod позволяет добавлять пользовательскую проверку через `.refine()`:

```
z.string().refine((val) => val.length > 3, {
  message: "Должно быть длиннее 3 символов",
});
```

Пример: проверка, что строка — это корректный URL с использованием конструктора `URL`:

```
z.string().refine((val) => {
  try {
    new URL(val);
    return true;
  } catch {
    return false;
  }
}, { message: "Должен быть корректным URL" });
```

___

### 4\. Вывод типов

Типы выводятся автоматически без дублирования:

```
const schema = z.object({ name: z.string() });
type SchemaType = z.infer<typeof schema>; 
// { name: string }
```

То есть, схема валидации становится единственным источником истины.

___

### 5\. Композиция

Схемы можно переиспользовать и расширять:

```
const base = z.object({ id: z.string() });
const user = base.extend({ name: z.string() });
```

Или делать их необязательными, допускающими `null` или со значением по умолчанию:

```
z.string().optional()
z.string().nullable()
z.string().default("Без названия")
```

## 🚀 Итог

Zod предоставляет:

-   Валидацию во время выполнения
-   Безопасность типов во время компиляции
-   Чистый и функциональный синтаксис
-   Единственный источник истины для структуры данных

Zod особенно мощен в TypeScript-приложениях полного стека, включая фреймворки вроде **Next.js**, **tRPC**, **Prisma** и др.
